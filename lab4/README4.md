Тема 1.4. Структуры данных. Работа в VS Code со структурами данных «деревья/графы»

1. Деревья 
-Анализ метода дерева для каждого яп:
 1)Python
 Код на Python отличается максимальной лаконичностью и читаемостью: динамическая типизация и встроенные структуры (словарь, кортеж) позволяют выразить алгоритм компактно. Использование модуля heapq даёт эффективную реализацию приоритетной очереди без излишней сложности. Гибкость синтаксиса и отсутствие явных объявлений типов ускоряют разработку, но снижают контроль над типами данных
 2)JAVA
 Код на Java демонстрирует строгую типизацию и объектно‑ориентированный подход: все переменные и коллекции требуют явного указания типа, а использование Map и PriorityQueue обеспечивает надёжность и предсказуемость. Шаблоны (Generics) и стандартные интерфейсы повышают безопасность типов, но увеличивают объём кода. Явное управление объектами и чёткая структура отражают корпоративный стиль разработки с акцентом на поддерживаемость и масштабируемость
 3)C++
   Код на C++ демонстрирует строгое типизирование и контроль ресурсов: все типы явно заданы, а STL‑контейнеры (map, priority_queue) обеспечивают баланс между удобством и производительностью. Приоритетная очередь настроена с компаратором greater, что гарантирует извлечение элемента с минимальным расстоянием. Синтаксическая громоздкость (явное объявление типов, сложные шаблоны) компенсируется прямым управлением поведением программы и эффективностью исполнения

2. Графы (DFS)
-Анализ метода графов для каждого яп:
   1)Python
   Реализация DFS на Python отличается предельной лаконичностью: используется динамическая типизация и встроенные структуры данных (словарь для графа, множество для отслеживания посещённых вершин). Рекурсивный подход естественным образом отражает логику обхода, а отсутствие явных объявлений типов упрощает код. Гибкость языка позволяет легко модифицировать действия при посещении вершин
   2)JAVA
   В Java реализация DFS демонстрирует строгую типизацию и объектно‑ориентированный подход: все коллекции (Map, List, Set) требуют явного указания типов. Управление памятью и проверка на null делают код более объёмным, но предсказуемым. Структура метода внутри класса и явная передача параметров подчёркивают корпоративный стиль разработки с акцентом на поддерживаемость
   3)C++
   Реализация на C++ сочетает высокую производительность (за счёт хеш‑таблиц unordered_map и unordered_set) с детальным контролем над ресурсами. Использование const и ссылок оптимизирует вызовы, а работа с итераторами и явными проверками обеспечивает точность. Синтаксическая насыщенность компенсируется возможностью тонкой настройки алгоритма под конкретные требования к скорости и памяти
